# 루루봇 (23. 09 - 24. 02)

https://rooroo.kr/

`Next.js` `Typescript` `GitHub` `CSS3` `Bootstrap` `Context API` `Fabric.js` `TTS` 

> 캔버스를 이용한 후원 꾸미기 및 스푼 방송 도움 프로그램
> 

### 구현사항

- 유튜브 검색 api를 통해 검색 목록 출력 후 iframe으로 영상 드러내기
- 후원 기록 데이터를 api로 받아와 스티커 형태로 미리보기 보여준 후, 드래그 앤 드롭으로 같은 형태로 캔버스에 표출
- 로컬서버에 업로드한 이미지를 미리보기로 보여준 후 드래그 앤 드롭하여 캔버스에 표출
- JSON 형태의 Lottie 이미지를 fetch 받아온 후, 재생바를 통해 움직이는 Lottie 이미지를 원하는 프레임에 멈추어 캔버스로 드래그 앤 드롭
- 캔버스 내에서 이미지 **크롭기능** [🔗](https://velog.io/@steela/Fabric.js-%EC%9D%B4%EB%AF%B8%EC%A7%80-%ED%81%AC%EB%A1%AD-Next.js-Typescript)
- 사용자의 모니터 크기를 고려하여 캔버스 줌인/줌아웃/움직이기 기능
- 캔버스 PNG 이미지로 고화질 다운로드

### 성과

- 기존 후원 꾸미기 시간 3-4시간에서 30-40분으로 단축

### `트러블슈팅`

1. 문제상황
    - Lottie 이미지가 캔버스에 드러나는 과정의 복잡합
        - Lottie 이미지(JSON)를 재생, 정지 버튼으로 프레임을 조정하여 원하는 타이밍의 정지 이미지를 캔버스로 드래그앤드롭.
        - JSON 파일이 캔버스 용량문제로 그대로 업로드되지 못함.
        - 드래그앤드롭하는 순간 svg 파일로 업로드 처리하여 로컬 서버에 저장한 후, 캔버스에 표출
    - 하지만 정지했을 때의 이미지가 캔버스에 표출되어야 하는데, 그 전 정지된 이미지가 표출되는 등 의도대로 구현되지 않음.
    - 정지되었을 때의 Lottie 이미지 프레임(JSON)이 즉시 svg 파일화하여 캔버스에 업로드 되어야하는데, svg 파일 업로드 과정에서 즉각 반영이 안됨.
    - 재생, 정지 버튼만으로 원하는 프레임 선택이 어려워 사용성 문제O.
- 문제해결
    - 재생/정지를 input type=range로 수정
        - 원하는 이미지 프레임 바로 선택 가능
    - Lottie 태그에 ref 및 스냅샷 버튼 추가
        - 해당 프레임을 정확하게 캐치한 후, svg 파일로 업로드하도록 로직 추가
    - svg 파일화 검증 로직 추가
        - 실패 시 사용자에게 다시 스냅샷 하도록 안내

# 요양시설 중개 플랫폼, 나으리 (23. 03 - 23.06)

`Next.js` `Typescript` `CSS` 
`HTML` `Javascript`  `GitHub` 
`Bootstrap` `카카오지도 API` `네이버지도 API` `Context API` 

> 공공데이터를 이용하여 요양시설 정보를 지도 검색 기능을 제공하는 반응형 요양 시설 중개 플랫폼
> 

### 구현사항

- 카카오 맵과 네이버 지도 api를 사용한 지도 및 요양시설 목록 페이지
- 성능 개선 이슈로 마이그레이션
    
    HTML + jQuery → HTML + Vanila Javascript → Next.js 12 (pages directory) → Next.js 13 (app directory)
    
- 관리자 페이지

### `트러블슈팅`

1. 문제 상황
    - HTML + jQuery 기술 사용 당시, 데이터를 불러오는 데 사용하는 api와 데이터의 양이 너무 많아 페이지 로딩 속도 저하
    - jQuery를 Vanila Javascript로 전부 변경 해도 느린 상황
- 문제 해결
    - Next.js 12로 마이그레이션
    - 추가 성능 개선을 위해 13으로 업그레이드

2. 문제 상황
    - 검색창 입력 후, 검색 목록 선택하려하면 검색 목록이 사라져서 선택 불가
    - 검색창 input에 focus 되면 검색 목록이 보이는데, 검색 목록을 클릭하는 순간 focus out 되어 목록창이 사라짐
    - 검색창이 focus 이벤트 발생했을 때 검색 목록창을 보여줌.
    - keyup, keydown 이벤트로 변경해봐도 동일한 결과.
- 문제 해결
    - 검색 목록창이 사라지는 조건을 검색창, 검색목록창, 목록창 안의 모든 요소들에 id 값을 주고, 해당 id 값 이외에 포커스 아웃되는 걸로 체크.
    
1. 문제 상황
    - 네이버 지도의 줌 레벨이 올라갈수록 고정해둔 마커 크기가 커서 시설의 이름이 겹쳐보여 사용성이 매우 저하됨
    - 마커 클러스터를 사용해 중복을 최소화 했어도 여전히 겹침
- 문제 해결
    - 줌레벨별로 마커의 스타일을 조정할 클래스를 추가 및 삭제해서 적용.
    - 줌레벨이 매우 낮은데도 겹치는 경우에는 커스텀 마커 오버레이로 시설 이름을 목록화하여 보여줌
